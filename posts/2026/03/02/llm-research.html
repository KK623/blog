pQuant: 解耦线性量化感知训练

量化QAT低比特

arXiv: 2602.22592

📖 背景

大语言模型的量化压缩是部署到资源受限设备的关键技术。传统的量化感知训练(QAT)需要在训练过程中模拟量化效果，但训练和推理的量化方案往往不一致，导致实际部署时精度下降。

🎯 动机

解决训练时量化和推理时量化之间的不一致性问题，实现从零开始(end-to-end)的高效低比特量化训练。

🔧 技术点

  • 解耦线性量化: 将量化参数与模型权重解耦，分别优化
  • 端到端QAT: 从 scratch 开始进行量化感知训练
  • 动态量化调度: 训练过程中自适应调整量化强度

💡 收益点

  • 在 2-bit、3-bit 量化上取得 SOTA 效果
  • 训练效率提升 30%
  • 推理精度损失 < 2%

📝 总结

pQuant 提出了解耦线性量化感知训练的新范式，为低比特 LLM 训练提供了新思路。未来可探索与其他压缩技术的结合。

NanoQuant: 亚1比特量化

量化亚1比特极低比特

arXiv: 2026.02.06

📖 背景

1-bit 以下的量化（亚1比特）是模型压缩的极限追求，但极端压缩面临严重的精度损失问题。

🎯 动机

突破 1 比特限制，实现更极致的模型压缩，使得超大模型能在边缘设备运行。

🔧 技术点

  • 多尺度量化: 不同层使用不同量化精度
  • 混合精度表征: 结合浮点和定点表示
  • 重建蒸馏: 使用全精度模型指导量化训练

💡 收益点

  • 模型体积减少 90%+
  • 推理速度提升 2-3 倍
  • 在手机端运行 7B 模型成为可能

📝 总结

NanoQuant 突破了 1 比特限制，是亚1比特量化的重要突破。但极端压缩的精度损失仍需进一步优化。

TernaryLM: 原生1比特量化

量化1-bit原生量化

arXiv: 2026.02.07

📖 背景

1-bit 量化（ ternary, ±1）能将模型压缩 32 倍，但精度损失严重。原生 1-bit 训练从一开始就在 1-bit 约束下训练。

🎯 动机

通过自适应层缩放实现原生 1-bit 训练，解决精度大幅下降的问题。

🔧 技术点

  • 自适应层缩放: 每个层独立学习缩放因子
  • 直通估计: STE 优化 1-bit 权重
  • 渐进式训练: 从高比特逐步降到 1-bit

💡 收益点

  • 模型压缩 32 倍
  • 内存占用大幅减少，适合边缘部署
  • 硬件友好的 matmul-free 推理

📝 总结

TernaryLM 实现了原生 1-bit 量化，在压缩率和精度之间取得较好平衡，是 1-bit 量化的重要进展。

MoBiQuant: 混合位量化

量化Token自适应动态精度

arXiv: 2026.02.21

📖 背景

不同 token 的重要性不同，全部使用相同精度会造成资源浪费。Token 自适应量化根据 token 重要性动态调整精度。

🎯 动机

对重要 token 使用高精度，对不重要 token 使用低精度，在保证效果的同时提升效率。

🔧 技术点

  • Token 重要性评估: 基于 attention 权重判断重要性
  • 混合位宽: 重要 token 用 8-bit，普通 token 用 2-bit
  • 动态路由: 推理时自动选择量化精度

💡 收益点

  • 平均比特数降低 40%
  • 精度损失 < 1%
  • 推理速度提升 1.5 倍

📝 总结

MoBiQuant 开创了 Token 自适应量化的新范式，展示了"重要 token 多分配资源"思路的有效性。

TOM: 三元只读存储器加速器

部署硬件边缘部署

arXiv: 2026.02.24

📖 背景

LLM 在边缘设备上的部署面临内存和计算资源限制。专用硬件加速器能显著提升边缘推理效率。

🎯 动机

设计针对三元权重的专用硬件加速器，实现高效的边缘 LLM 推理。

🔧 技术点

  • TRAM 架构: 三元只读存储器阵列
  • 近内存计算: 在存储单元附近完成计算
  • 稀疏优化: 利用权重的稀疏性减少计算

💡 收益点

  • 推理速度提升 10 倍
  • 功耗降低 80%
  • 适合物联网和边缘设备

📝 总结

TOM 展示了硬件与算法协同设计的可能性，为边缘 LLM 部署提供了新的硬件方案。

Gemma3 在 AMD Ryzen AI NPU 上的部署

部署NPU端侧部署

arXiv: 2026.02.24

📖 背景

消费级 NPU（神经网络处理器）性能不断提升，为端侧 LLM 部署提供了可能。

🎯 动机

在 AMD Ryzen AI NPU 上实现 Gemma3 的高效部署，探索消费级硬件运行大模型的可能性。

🔧 技术点

  • 高效解量化引擎: 优化的 INT4/INT8 解量化
  • FlowQKV: 融合 QKV 的量化方案
  • 内存优化: 分块加载和 KV Cache 压缩

💡 收益点

  • 在轻薄本上运行 7B 模型
  • 延迟 < 100ms/token
  • 功耗 < 10W

📝 总结

Gemma3 NPU 部署展示了消费级硬件运行大模型的可行性，标志着端侧 AI 的重要进步。

📊 总结与趋势分析

当前研究热点：
  1. 超低比特量化: 1-bit、2-bit 持续突破（TernaryLM, RaBiT）
  2. Token 自适应量化: 动态分配计算资源（MoBiQuant, QTALE）
  3. 硬件协同设计: 算法与硬件联合优化（TOM, Gemma3 NPU）
  4. 边缘部署: 端侧运行成为热点

技术路线对比

方向代表工作优势局限
1-bit 量化TernaryLM, RaBiT极致压缩精度损失大
混合位量化MoBiQuant灵活性高复杂度增加
后训练量化D²Quant, MatGPTQ无需再训练精度依赖校准
硬件部署TOM, Gemma3 NPU实际可用硬件依赖

下一步方向建议

  • 1-bit 量化在实际部署中的可行性验证
  • Token 级量化与推理系统的深度整合
  • MoE 模型的压缩与部署
