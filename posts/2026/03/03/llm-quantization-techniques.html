011. QLoRA: 量化微调的里程碑

arXiv: 2305.14314

QLoRA (Quantized LoRA) 是华盛顿大学团队提出的革命性方法，首次实现在消费级GPU上微调65B参数的大语言模型。其核心创新是将4位量化与LoRA微调相结合，开启了"量化微调"的新范式。

1.1 NF4 量化公式 (NormalFloat 4-bit)

NF4是QLoRA的核心数据类型，专门为正态分布的权重设计。传统INT4量化将权重均匀映射到16个离散值，而NF4根据正态分布的非均匀分位数进行量化，能够更好地保留权重信息。

NF4 量化公式：

quantized = round( (w - w_min) / (w_max - w_min) × (2^b - 1) )

其中 b = 4 表示4位量化，2^b = 16 个量化级别。NF4的关键是使用分位数(Quantile)而非均匀划分来定义量化网格：

quantile_i = (i + 0.5) / 2^b, for i ∈ [0, 2^b - 1]

具体实现中，NF4使用双佛雷塞(Quantile Quantization)方法：对权重分布进行分位数计算，得到16个非均匀的量化中心；量化中心根据正态分布的累积分布函数(CDF)确定；这样量化后的值在统计上更接近原始分布。

1.2 双重量化 (Double Quantization)

双重量化是QLoRA的第二个关键创新。量化不仅应用于权重，还应用于量化过程中产生的缩放因子(Scale Factor)，进一步压缩内存。

双重量化原理：

第一层量化：
w_quant = round(w / s) × s
其中 s 是缩放因子，s = (w_max - w_min) / (2^b - 1)

第二层量化：
s_quant = round(s / s_c) × s_c
其中 s_c 是用于量化缩放因子的常量
内存节省效果： 原始方式每参数需要 0.4 bits 存储缩放因子，双重量化后降至 0.2 bits，节省约50%内存

1.3 分页优化器 (Paged Optimizers)

训练过程中的优化器状态(如Adam的动量和方差)是内存的主要消耗之一。QLoRA使用CPU内存来存储这些状态，避免GPU显存溢出。

分页优化器架构

GPU显存:
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│ 4-bit权重   │  │ 16-bit LoRA │  │ 16-bit梯度  │
└─────────────┘  └─────────────┘  └─────────────┘
       ↓ (优化器状态卸载)
CPU内存:
┌─────────────────────────────┐
│ Adam动量 m_t, 方差 v_t      │
│ 按需分页调入GPU              │
└─────────────────────────────┘

工作流程： (1) 梯度计算在GPU上完成 (2) 优化器状态卸载到CPU (3) 需要更新时从CPU加载状态 (4) 在GPU上完成参数更新

1.4 QLoRA 整体架构

QLoRA 整体数据流

输入 x
   ↓
┌─────────────────────────────────┐
│  冻结的NF4量化基础模型          │
│  LayerNorm→QKV→Attention→MLP   │
└─────────────────────────────────┘
   ↓ (临时反量化)
┌─────────────────────────────────┐
│  16-bit LoRA适配器             │
│  ΔW = A×B (rank=64)           │
└─────────────────────────────────┘
   ↓
输出 = 基础模型 + LoRA输出
QLoRA 关键成果： 65B参数模型可在单张48GB GPU上微调；Vicuna基准达到ChatGPT性能的99.3%；内存降低10倍以上

022. GPTQ: 基于二阶信息的精确量化

arXiv: 2210.17323

GPTQ (Generative Pre-trained Transformer Quantization) 由维也纳大学团队提出，是后训练量化(Post-Training Quantization, PTQ)领域的里程碑工作。它首次实现了在4小时内将175B参数模型量化到3-4位，且精度损失极小。

2.1 Hessian 矩阵近似

GPTQ的核心洞见是：不同权重对量化的敏感度不同。敏感度高的权重量化后对模型输出影响大，需要更精细的量化；敏感度低的权重可以粗量化。

Hessian 矩阵定义：

H = ∇²L(w) = ∂²L / ∂w²

其中 L 是损失函数，w 是权重向量。

对角近似：
H_ii ≈ ∂²L / ∂w_i²

量化第i个权重时的敏感性：
sensitivity_i = H_ii × (error_i)²

由于完整计算Hessian矩阵O(n²)复杂度太高，GPTQ使用对角Hessian近似来降低计算复杂度。

2.2 二阶信息量化公式

GPTQ使用最优脑量化(Optimal Brain Quantization, OBQ)框架的改进版本，通过二阶信息来指导量化：

GPTQ 量化目标函数：

minimize: ||W·X - Ŵ·X||²

其中 Ŵ 是量化后的权重矩阵，X 是输入激活。

等价于最小化：
minimize: tr((W - Ŵ)ᵀ H (W - Ŵ))

权重更新公式：
w_i_new = w_i - H_ii⁻¹ × e_i
其中 e_i 是量化误差，H_ii 是Hessian对角元素

算法流程： (1) 计算每个权重列的Hessian对角近似 (2) 按敏感性排序，从低到高依次量化 (3) 量化当前权重，计算误差 (4) 将误差通过Hessian逆加权后分配给未量化权重 (5) 重复直到所有权重量化完成

2.3 误差补偿机制

GPTQ的关键创新是误差补偿：量化一个权重产生的误差会累积，并通过Hessian加权分配给后续权重。

误差传播公式：

Δw_j = - (H_jj)⁻¹ × H_ij × Δw_i

其中 Δw_i 是第i个权重的量化误差，Δw_j 是分配给第j个权重的补偿

GPTQ 逐列量化流程

输入权重 W (n × d)
   ↓
┌─────────────────────────────┐
│ H_diag = 2XXᵀ + λI         │
│ (校准数据集计算)            │
└─────────────────────────────┘
   ↓
┌─────────────────────────────┐
│ 逐列循环量化:                │
│ 1. w_i → ŵ_i (量化)        │
│ 2. e = w_i - ŵ_i (误差)    │
│ 3. Δw = -H_ii⁻¹ × H_i × e  │
│ 4. w_j += Δw_j (补偿)       │
└─────────────────────────────┘
   ↓
量化权重 Ŵ
GPTQ 关键成果： 175B参数模型仅需4 GPU小时量化；3-4位量化精度损失小于1%；A100上3.25倍推理加速

033. AWQ: 激活感知的权重保护

arXiv: 2306.00978

AWQ (Activation-aware Weight Quantization) 由MIT和微软团队提出，荣获MLSys 2024最佳论文奖。其核心洞见是：LLM中只有约1%的权重是关键权重，保护这些权重可以大幅降低量化误差。

3.1 通道保护机制 (Channel Protection)

AWQ发现权重的重要性应该由激活值分布决定，而不是权重分布。与大激活值相乘的权重即使数值不大，对输出的影响也很大。

量化误差分析：

对于线性层 y = Wx，量化误差为：
Δy = ΔW × x

均方误差：
E[||Δy||²] = E[||ΔW||²] × E[||x||²]

关键发现：与较大激活值相乘的权重应该更精细地量化

AWQ使用通道级缩放来保护重要通道：

通道保护公式：

步骤1 - 激活统计计算：
s_i = mean(|x_i|) (对每个输入通道计算平均绝对激活)

步骤2 - 缩放因子计算：
scale_i = s_i^α (α是保护强度超参数，通常α=0.5)

步骤3 - 缩放后量化：
w'_i = w_i / scale_i (缩放权重)
w_q = round(w'_i / s) × s (量化)
w_final = w_q × scale_i (反缩放)

3.2 SmoothQuant 平滑机制

AWQ提出了SmoothQuant技术来解决激活异常值问题。LLM的激活值中存在少数极大的异常值，这些值会导致整体量化精度下降。

SmoothQuant 平滑公式：

目标：将量化难度从权重转移到激活

W' = W × diag(s)^(-1)
x' = diag(s) × x

其中 s = (|W|_1 / n)^α × (|x|_1 / m)^(1-α)

关键参数： α 控制权重和激活之间的平衡；α=0.5 通常效果最好

AWQ 量化流程

输入: W, x, 校准数据
   ↓
┌─────────────────────────────┐
│ s_channel = mean(|x|)      │
└─────────────────────────────┘
   ↓
┌─────────────────────────────┐
│ scale = (s_channel ^ α)   │
└─────────────────────────────┘
   ↓
┌─────────────────────────────┐
│ W_smooth = W / scale       │
│ x_smooth = x × scale       │
└─────────────────────────────┘
   ↓
┌─────────────────────────────┐
│ W_q = quant(W_smooth)      │
└─────────────────────────────┘
   ↓
输出: W_q + scale
AWQ 关键成果： 4位量化下精度损失极小；桌面和移动GPU上比FP16快3倍以上；首次在移动GPU上运行70B Llama-2

044. SmoothQuant: 激活平滑的数学基础

arXiv: 2308.09886

SmoothQuant 是AWQ的核心技术创新，专门解决LLM中激活异常值问题。SmoothQuant的数学基础非常优雅，它通过数学推导证明了为什么将量化难度从权重转移到激活是有效的。

4.1 激活平滑公式详解

SmoothQuant的核心思想是：LLM的权重分布通常比较均匀，但激活值存在显著的通道间差异，某些通道的激活值远大于其他通道。

SmoothQuant 激活平滑公式：

线性层前向传播：
y = Wx + b

平滑变换：
W' = W × S^(-1)
x' = S × x
y = Wx = W'x'

其中 S = diag(s_1, s_2, ..., s_k) 是通道级缩放矩阵

缩放因子s的计算是SmoothQuant的关键：

缩放因子计算：

s_j = (|W|_F,j / n)^α × (|x|_F,j / m)^(1-α)

参数解释：
|W|_F,j：权重矩阵第j列的Frobenius范数
|x|_F,j：激活矩阵第j列的Frobenius范数
n：输出通道数；m：输入通道数
α：平衡因子，通常取0.5

4.2 为什么平滑有效？

SmoothQuant的有效性基于以下数学洞见：

量化误差分析：

原始量化误差：
Err_orig = ||W × x - Q(W) × x||

平滑后量化误差：
Err_smooth = ||W'S × S^(-1)x - Q(W'S) × S^(-1)x||

关键性质：
Err_smooth ≤ max(s_j) / min(s_j) × Err_orig

通过合理选择s，可以使 max(s_j)/min(s_j) 接近1

4.3 SmoothQuant 完整算法

SmoothQuant vs 标准量化

标准量化 (per-tensor):
  W: [d_in×d_out] → INT4
  x: [batch×d_in]  → INT8
  y = W×x

SmoothQuant (per-channel + smooth):
  s_j = (||W[:,j]||₂/n)^α × (||x[:,j]||₂/m)^(1-α)
  W' = W / s,  x' = x × s
  y = quant(W') × quant(x')
SmoothQuant 关键成果： 8位量化可达到FP16精度；3位量化精度损失小于2%；为AWQ提供数学基础

055. SpQR: 离群值感知的稀疏量化

arXiv: 2306.03078

SpQR (Sparse-Quantized Representation) 是另一种针对LLM离群值的创新方法。它通过识别和保护离群值，将大部分权重进行极端量化，同时保持离群值的精度。

5.1 离群值检测机制

SpQR观察到LLM权重中存在少数离群值：这些权重的绝对值远大于其他权重，如果用低比特量化这些离群值会产生巨大误差。

离群值检测公式：

基于标准差的检测：
|w_i| > μ + k × σ
其中 μ 是均值，σ 是标准差，k 是阈值(通常k=3)

基于百分位的检测：
|w_i| > percentile(|W|, p)
通常 p = 99.9 或 p = 99.99

5.2 混合精度量化架构

SpQR采用混合精度策略：对离群值使用高精度，对普通值使用低精度。

SpQR 混合精度存储

权重矩阵 W
   ↓
┌─────────────────────────────────────┐
│ 离群值检测: |w| > μ + 3σ           │
│ 结果:                               │
│  离群值 (~1-2%): 16-bit FP16       │
│  普通值 (~98%): 3-bit 量化         │
└─────────────────────────────────────┘
   ↓
存储格式:
  [索引表: 0.1%] [3-bit权重: 2.9] [16-bit离群值: 0.2]
总平均: ~3.3 bits/权重

5.3 量化公式与误差分析

SpQR 量化公式：

普通值量化：
w_normal_q = round(w / s) × s
其中 s = max(|W_normal|) / (2^(b-1) - 1)

离群值存储：
w_outlier = w (保持FP16精度)

重构输出：
y = W_normal_q × x + W_outlier × x

5.4 量化感知微调

SpQR还引入了量化感知微调来进一步减少量化误差：训练目标是最小化量化后的输出与全精度输出的KL散度；更新方式仅更新离群值权重，其他权重保持冻结。

SpQR 关键成果： 3位量化达到SOTA精度；通过离群值保护实现极端量化；首次实现175B模型的单GPU推理

066. BitNet: 1-bit Transformer 的开创者

arXiv: 2310.11453

BitNet 是微软研究院提出的1-bit Transformer架构，开创了LLM极端量化的先河。它将权重完全量化为{-1, +1}，实现了极致的压缩率和计算效率。

6.1 BitLinear 层的核心公式

BitNet的核心是BitLinear层，它将权重量化为二值{-1, +1}，同时通过缩放因子保持输出分布。

BitNet 量化公式：

权重二值化：
W_b = Sign(W) = { +1 if w ≥ 0, -1 if w < 0 }

缩放因子计算：
α = mean(|W|) (L1范数的均值)

前向传播：
y = α × W_b × x

其中 α 是缩放因子，补偿二值化带来的信息损失

6.2 训练时的直通估计器 (STE)

由于Sign函数不可导，BitNet使用直通估计器(Straight-Through Estimator, STE)来训练：前向传播使用二值权重，反向传播时梯度直接通过。

STE 前向/反向传播：

前向传播：
Ŵ = Sign(W) (二值化)

反向传播：
∂L/∂W = ∂L/∂Ŵ (梯度直通)

关键： 在反向时忽略Sign的符号函数性质

6.3 BitNet 架构设计

BitNet 1-bit Transformer 架构

标准Transformer:
  Input → LayerNorm → QKV → Attention → Dropout → Output

BitNet:
  Input → LayerNorm → BitLinear(Q) → BitLinear(K) → BitLinear(V)
                    → BitLinear(Attention) → BitLinear(Output)
                    → Dropout → Add → LayerNorm → FFN → Output

关键特点:
  • 所有Linear层替换为BitLinear
  • 权重存储: 1-bit (仅为原来的1/16)
  • 矩阵乘法: INT8 GEMM + Bit unpacking
  • 激活保持高精度 (8-16 bit)

6.4 量化与训练技巧

BitNet的训练使用了多项关键技巧：

  • 初始化：使用He初始化，确保初始权重方差合理
  • 学习率调度：使用warmup和余弦退火
  • 权重衰减：L2正则化防止权重漂移过大
  • 梯度裁剪：防止梯度爆炸
BitNet 训练损失：

与标准Transformer相同，使用交叉熵损失：

L = -Σ y_i × log(ŷ_i)

但梯度通过STE反向传播到二值权重
