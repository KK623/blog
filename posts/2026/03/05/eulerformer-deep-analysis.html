2. 研究背景

2.1 序列用户行为建模的重要性

在推荐系统中，用户行为序列（如点击历史、购买记录）包含了丰富的用户偏好信息。序列推荐任务旨在根据用户的历史行为预测其下一步可能感兴趣的项目。准确的序列建模能够：

  • 捕捉用户兴趣的演化：用户的偏好会随时间动态变化
  • 发现序列模式：某些项目之间存在顺序依赖关系
  • 提供个性化推荐：基于用户独特的历史行为生成推荐

2.2 Transformer 在序列推荐中的应用

Transformer 模型凭借其强大的序列建模能力，已成为序列推荐的主流方法。其核心是自注意力机制（Self-Attention），能够计算序列中任意两个位置之间的依赖关系。

自注意力的基本形式：

Attention(Q, K, V) = softmax(QK^T / √d_k) · V

其中 Q、K、V 分别是查询（Query）、键（Key）、值（Value）矩阵。

2.3 现有方法的核心问题

在 Transformer 中，注意力分数由两部分组成：

  • 语义差异（Semantic Difference）： 反映两个 token 在语义上的相似度，通过 Q·K^T 计算
  • 位置差异（Positional Difference）： 反映两个 token 在序列中的相对位置关系
核心问题： 现有方法通常以不同的方式建模语义差异和位置差异，这种分离建模可能限制了序列建模的表达能力。

2.4 现有位置编码方法的局限

(1) 绝对位置编码（Absolute Positional Encoding）：

  • 如原始 Transformer 的正弦/余弦位置编码
  • 将位置信息添加到输入 embedding 中
  • 问题：语义和位置信息在输入层混合，难以在注意力层灵活控制

(2) 相对位置编码（Relative Positional Encoding）：

  • 如 Transformer-XL、T5、DeBERTa 等方法
  • 在注意力计算中显式引入相对位置信息
  • 问题：通常只修改注意力分数的计算方式，与语义信息分离处理

(3) 旋转位置编码（RoPE - Rotary Position Embedding）：

  • 如 LLaMA、PaLM 等大模型使用
  • 利用复数旋转将位置信息编码到 query/key 中
  • 问题：只处理位置信息，语义差异仍用点积计算，两者未统一

3. 研究动机

3.1 核心洞察

作者观察到，语义差异和位置差异本质上是同一类问题——都是两个向量之间的"差异"。为什么不能在一个统一的框架中建模这两种差异呢？

核心问题： 如何在统一的理论框架中同时建模语义差异和位置差异，使得两者能够相互增强而非相互独立？

3.2 从 EulerNet 到 EulerFormer

EulerNet 在 CTR 预测中成功应用了复数向量表示和欧拉公式。作者思考：能否将这种思想扩展到 Transformer 的注意力机制中？

  • 语义角度（Semantic Angle）： 用复数向量的相位表示两个 token 的语义关系
  • 位置角度（Positional Angle）： 用复数向量的相位表示两个 token 的位置关系
  • 统一表示： 总相位 = 语义相位 + 位置相位，通过欧拉公式统一编码

3.3 创新目标

(1) 统一理论框架： 提供一个能够同时处理语义和位置的统一数学框架。

(2) 灵活控制机制： 能够根据语义上下文自适应地调整语义和位置的融合程度。

(3) 改进表示质量： 通过特殊的训练任务改进上下文表示的各向同性（isotropy）。

(4) 完备性与通用性： 理论框架应具有高度完备性，能够涵盖现有方法作为特例。

4. 核心技术点

4.1 复数向量注意力机制

EulerFormer 的核心创新是将注意力机制扩展到复数向量空间。

复数表示： 每个 token 的 query/key/value 都表示为复数向量：

z = r · e^(iθ) = r(cos θ + i sin θ)

其中 r 是模（magnitude），θ 是相位（phase/angle）。

4.2 欧拉变换函数（Euler Transformation）

关键创新： 提出了一种新的变换函数，将实数序列 token 转换为极坐标形式的复数向量。

变换过程：
1. 将语义 embedding 转换为语义角度 θ_s
2. 将位置编码转换为位置角度 θ_p
3. 总角度 θ = θ_s + θ_p（或自适应融合）
4. 通过欧拉公式生成复数向量：z = r · e^(iθ)

这种变换使得语义信息和位置信息能够在复数空间中统一表示。

4.3 差分旋转机制（Differential Rotation Mechanism）

这是 EulerFormer 的另一核心创新。传统的位置编码通常固定位置角度，而 EulerFormer 允许根据语义上下文自适应地调整语义角度。

自适应函数（Adaptation Function）：

θ_total = Adapt(θ_semantic) + θ_positional

其中 Adapt(·) 是一个可学习的函数，能够根据当前语义上下文动态调整语义角度的权重。

优势：

  • 某些场景下语义更重要（如短序列），可以增加语义角度的权重
  • 某些场景下位置更重要（如长序列），可以增加位置角度的权重
  • 实现了语义和位置的动态平衡

4.4 复数注意力计算

在复数空间中，注意力分数的计算方式也相应改变：

Attention(Q, K, V) = softmax(Re(Q · K^* / √d_k)) · V

其中 K^* 表示 K 的共轭复数，Re(·) 表示取实部。

复数乘法天然编码了角度相加：

z₁ · z₂ = r₁r₂ · e^(i(θ₁+θ₂))

这意味着两个 token 的语义角度和位置角度会自动相加，实现了统一建模！

4.5 相位对比学习（Phase Contrastive Learning）

为了改进表示的各向同性（isotropy），EulerFormer 提出了相位对比学习任务。

问题背景： 传统 Transformer 的表示往往存在各向异性（anisotropy）问题，即向量分布在狭窄的锥形区域内，影响表达能力。

解决方案：

  • 将同一序列的不同增强视图作为正样本对
  • 最大化正样本对的相位相似度
  • 最小化负样本对的相位相似度
  • 鼓励表示在复数空间中更均匀分布

效果： 实验表明相位对比学习显著提升了表示质量和下游任务性能。

4.6 理论完备性

EulerFormer 的理论框架具有以下性质：

  • 完备性（Completeness）： 能够表示所有类型的语义-位置关系
  • 通用性（Generality）： 现有方法（如 RoPE）可以看作是 EulerFormer 的特例
  • 鲁棒性（Robustness）： 对语义变化更加鲁棒
  • 优越性（Superiority）： 在理论上具有更优的性质（如更好的梯度流）

5. 实验结果

5.1 数据集

实验在四个公开数据集上进行：

数据集 类型 用户数量 项目数量 交互数量
Amazon-Beauty 电商 22K 12K 0.2M
Amazon-Sports 电商 35K 18K 0.3M
Yelp 点评 45K 11K 0.7M
MovieLens-1M 电影 6K 3.7K 1M

5.2 主要性能对比

Recall@10 指标（越高越好）：

模型 Beauty Sports Yelp MovieLens
SASRec 0.0825 0.0612 0.0723 0.2341
BERT4Rec 0.0798 0.0594 0.0698 0.2289
FMLP-Rec 0.0841 0.0628 0.0745 0.2389
LightSANs 0.0839 0.0631 0.0752 0.2412
EulerFormer (本文) 0.0923 0.0712 0.0834 0.2567

结论： EulerFormer 在所有四个数据集上都显著优于现有方法，Recall@10 提升约 6%~12%。

5.3 消融实验

作者进行了详细的消融实验，验证了各组件的有效性：

  • 去除差分旋转机制： Recall@10 下降约 3%，说明自适应调整的重要性
  • 去除相位对比学习： Recall@10 下降约 2%，说明表示质量改进的有效性
  • 去除复数表示（改用实数）： Recall@10 下降约 5%，说明复数表示的核心作用
  • 固定位置权重： 性能下降，说明自适应融合的必要性

5.4 与 RoPE 的对比

RoPE（旋转位置编码）是 LLaMA 等大模型使用的位置编码方法。作者对比了 EulerFormer 和 RoPE：

  • RoPE： 只处理位置信息，语义差异仍用点积计算
  • EulerFormer： 同时处理语义和位置，统一建模
  • 结果： EulerFormer 在所有数据集上均优于 RoPE，验证了统一框架的优势

6. 收益点

6.1 性能提升

  • 在四个标准数据集上均取得 SOTA 性能，Recall@10 提升 6%~12%
  • NDCG 指标同样优于对比方法，验证了排名质量
  • 在不同类型的序列推荐任务（电商、点评、电影）上都表现优异

6.2 理论创新

核心贡献： 首次提出了统一建模语义差异和位置差异的理论框架，填补了该领域的理论空白。
  • 完备性： 理论框架高度完备，能够涵盖现有方法作为特例
  • 通用性： 不仅适用于推荐系统，也可扩展到 NLP、CV 等其他领域
  • 鲁棒性： 对语义变化更加鲁棒，在长序列上表现更稳定

6.3 表示质量改进

  • 各向同性（Isotropy）： 通过相位对比学习显著改进了表示的各向同性
  • 语义分离度： 不同语义的表示在复数空间中分离更清晰
  • 可解释性： 相位信息具有直观含义，便于理解模型决策

6.4 自适应能力

  • 动态平衡： 能够根据语义上下文自适应调整语义和位置的权重
  • 场景适应： 不同场景下自动选择最优的融合策略
  • 无需调参： 减少了手动调整位置编码超参数的工作量

6.5 计算效率

  • 额外开销小： 复数运算的额外计算开销很小（约 5%）
  • 内存友好： 复数表示不会显著增加内存占用
  • 易于实现： 基于 PyTorch 的复数支持，实现简单

7. 局限性与未来工作

7.1 当前局限性

(1) 硬件支持有限：

  • 复数运算在 GPU/TPU 上的优化不如实数运算成熟
  • 大规模训练时可能遇到效率瓶颈

(2) 超参数敏感性：

  • 自适应函数的架构需要一定调参
  • 不同数据集上最优配置可能有差异

(3) 长序列扩展性：

  • 在极长序列（如文档级文本）上的效果有待验证
  • 与线性注意力等高效方法的结合需要探索

7.2 未来研究方向(1) 大语言模型应用：

  • 将 EulerFormer 的思想应用到 LLM 的预训练中
  • 探索复数注意力在超大模型上的效果
  • 结合 MoE（混合专家）等架构进一步优化

(2) 多模态扩展：

  • 将复数表示扩展到图像、音频等多模态数据
  • 探索跨模态的复数注意力机制

(3) 理论深入研究：

  • 深入分析复数注意力的表达能力上界
  • 探索其他数学结构（如四元数）在注意力中的应用
  • 研究复数表示与神经网络泛化能力的关系

(4) 硬件优化：

  • 设计专门针对复数运算的硬件加速器
  • 优化复数矩阵乘法的 CUDA kernel

8. 总结

EulerFormer 是 SIGIR 2024 上的重要工作，它成功地将 EulerNet 中的复数向量思想扩展到 Transformer 的注意力机制中，开创了统一建模语义和位置的全新范式。

核心贡献回顾：

  1. 统一框架： 首次提出统一建模语义差异和位置差异的理论框架，具有高度完备性和通用性
  2. 技术创新： 提出欧拉变换函数和差分旋转机制，实现语义和位置的自适应融合
  3. 表示改进： 通过相位对比学习显著改进了上下文表示的各向同性
  4. 性能领先： 在四个标准数据集上显著优于现有方法，验证了方法的有效性
  5. 理论完备： 理论分析证明了框架的完备性、鲁棒性和优越性

与 EulerNet 的对比：

特性 EulerNet (SIGIR 2023) EulerFormer (SIGIR 2024)
任务 CTR预测 序列推荐
架构 特征交互网络 Transformer
核心创新 特征交互的欧拉公式 注意力的复数向量扩展
统一建模 显式+隐式交互 语义+位置差异

对 LLM 研究的启示：

EulerFormer 对当前大语言模型研究有重要启示：

  • RoPE 的扩展： 当前 LLaMA、Qwen 等模型使用 RoPE，EulerFormer 提供了进一步改进的方向
  • 统一建模趋势： 语义和位置的统一建模可能是 Transformer 架构的下一个演进方向
  • 复数神经网络： 复数表示在注意力中的成功应用，为探索复数 LLM 提供了理论基础
  • 表示质量： 相位对比学习改进各向同性的思想，可以应用于 LLM 的表示学习
总体评价： EulerFormer 是一篇将数学理论与深度学习精妙结合的典范工作。它不仅推进了序列推荐领域的发展，更为 Transformer 架构的演进提供了新的思路。其与 EulerNet 形成的"复数神经网络"研究线，展示了中国人民大学 RUCAIBox 团队在这一方向的持续创新和深厚积累。
