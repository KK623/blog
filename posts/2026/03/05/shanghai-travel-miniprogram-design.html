🗺️ 上海周边智能旅游小程序

产品设计文档 | 基于时间和预算的智能推荐系统

文档版本： v1.0 | 创建时间：2026-03-05 12:35
设计者：Jarvis & K

1. 项目概述

产品定位： 面向上海本地居民的智能旅游推荐小程序，结合时间、预算和个人偏好，推荐上海周边旅游地点、酒店和交通方案。

1.1 核心价值主张

  • 省时： 输入想法即可获得完整旅游方案
  • 省心： 集成小红书真实笔记，了解他人体验
  • 智能： AI 根据预算和时间自动规划最优路线

1.2 目标用户

用户群体 需求特征 使用场景
上海本地居民 周末短途游、追求效率、熟悉周边 "周末想去户外，预算500，推荐个地方"
特定人群 亲子、情侣、宠物主、户外爱好者 "带3岁宝宝，车程1小时内，有儿童设施"
设计决策： 采用分 sheet 设计，为两类用户提供差异化体验。

2. 功能设计

2.1 功能优先级

优先级 功能模块 说明 状态
P0 地点收录 上海周边旅游地点展示（图文介绍） 待开发
P0 智能推荐 基于时间+预算+想法的 AI 推荐 待开发
P1 小红书笔记 关联地点的小红书内容展示 待开发
P1 交通规划 路线规划（驾车/公交/步行） 待开发
P2 酒店推荐 仅推荐，不含预订功能 待开发

2.2 核心功能详解

智能推荐逻辑（混合方案）：

  • 步骤1 - 规则筛选： 根据预算、时间、距离等硬性条件过滤地点
  • 步骤2 - AI 生成： 调用 GPT API 生成推荐理由和完整方案
  • 步骤3 - 结果组装： 整合地点信息 + 酒店推荐 + 交通路线
示例交互：
用户："周末想带狗去户外，预算500，住在浦东"
系统：推荐宠物友好的郊野公园 → 生成包含自驾路线的完整方案 → 推荐附近宠物友好酒店

3. 技术架构

3.1 技术选型

选定方案：方案 A（微信小程序原生 + 微信云开发）
理由：开发快速、无需服务器运维、微信生态无缝集成
层级 技术/服务 说明
前端 微信小程序原生框架 WXML + WXSS + JavaScript
后端 微信云开发 云函数 + 云数据库 + 云存储
地图服务 高德/百度地图 SDK 地点搜索、路径规划、导航
AI 服务 GPT API 智能推荐生成
内容源 小红书 API 笔记内容展示（需申请企业资质）

4. 数据源规划

数据类型 数据源 获取方式 难度
旅游地点 高德/百度地图 POI API 官方 API ⭐ 较易
小红书笔记 小红书开放平台 需企业资质申请 ⭐⭐⭐ 较难
酒店信息 携程/美团 API 需商务合作 ⭐⭐ 中等
交通规划 高德/百度地图路径规划 官方 API ⭐ 较易
风险预案：
• 小红书 API 如申请困难，可采用替代方案（用户粘贴链接 + 摘要展示）
• 酒店信息可用高德地图酒店 POI 作为备选
• 所有 API 方案先申请测试，正式商用前需确认合规性

5. 页面结构设计

5.1 Tab 导航（底部）

Tab 功能 主要内容
🏠 首页 智能推荐入口 搜索框 + 快捷标签（周末/亲子/户外等）+ 推荐卡片
⭐ 推荐 个性化推荐 基于历史行为的推荐列表 + 季节/主题活动
📍 地点 地点大全 分类浏览（自然/人文/美食等）+ 地图模式 + 收藏
👤 我的 个人中心 历史记录 + 收藏夹 + 设置 + 反馈

5.2 关键页面流程

推荐生成流程：

  1. 用户在首页输入想法（如"周末带狗去户外，预算500"）
  2. 系统解析需求（时间、预算、偏好、位置）
  3. 规则引擎筛选符合条件的地点
  4. AI 生成推荐理由和完整方案
  5. 展示结果：地点详情 + 小红书笔记 + 酒店推荐 + 交通路线
  6. 用户可收藏、分享或重新生成

6. 技术实现要点

6.1 数据库设计（云数据库）

集合：spots（地点） - _id: 地点ID - name: 名称 - category: 分类（自然/人文/美食/户外等） - location: 坐标 {latitude, longitude} - address: 地址 - tags: 标签数组（宠物友好/亲子/免费等） - images: 图片URL数组 - description: 描述 - price_range: 价格区间 - open_hours: 开放时间 - rating: 评分 - xiaohongshu_notes: 关联笔记ID数组 集合：recommendations（推荐记录） - _id: 记录ID - user_id: 用户ID - input: 用户输入文本 - parsed_intent: 解析后的意图 - results: 推荐结果数组 - created_at: 创建时间

6.2 GPT Prompt 设计

推荐生成 Prompt 框架：
你是一个上海周边旅游专家。根据以下信息为用户推荐旅游方案：
• 用户位置：{location}
• 可用时间：{time}
• 预算：{budget}
• 特殊需求：{preferences}
• 候选地点：{spots_list}

请输出：1.推荐地点及理由 2.交通建议 3.酒店推荐 4.注意事项

7. 开发计划

7.1 MVP 阶段（4-6周）

周次 任务 交付物
第1周 环境搭建 + 数据收集 小程序框架 + 50个地点数据
第2周 地点展示功能 地点列表 + 详情页
第3-4周 智能推荐功能 推荐算法 + GPT集成
第5周 地图 + 交通规划 地图SDK集成 + 路线规划
第6周 测试优化 + 上线 MVP版本发布

7.2 后续迭代

  • V1.1： 小红书笔记集成、用户评论功能
  • V1.2： 酒店预订跳转、优惠券功能
  • V2.0： 社交功能、行程分享、结伴出游

8. 风险评估

风险 影响 应对方案
小红书 API 申请失败 高 采用用户UGC模式，用户手动粘贴笔记链接
GPT API 费用过高 中 缓存推荐结果，限制每日免费推荐次数
地点数据不足 中 优先覆盖热门地点，逐步扩充
用户冷启动 低 提供热门推荐模板，无需输入即可获得方案
